# 📘 文章系统前后端对接文档

## 1. 文档说明
本文档用于指导前端开发人员对接文章系统后端接口。

系统特性：
- 基于 RESTful 风格设计
- 统一 JSON 请求 / 响应
- 使用 JWT 认证
- ID 类型统一为 String
- 时间字段使用 LocalDateTime
- 用户 / 文章 / 评论为软删除
- 点赞 / 收藏为硬删除

---

## 2. 基础约定

### 2.1 通用请求头
| 字段 | 必填 | 说明 |
|------|------|------|
| Content-Type | 是 | application/json |
| Authorization | 需要登录接口必填 | Bearer {token} |

说明：
- 需要登录的接口必须携带 Authorization  
- 后端从 token 中解析 `userId` 与 `role`  
- 前端无需传 `userId`

---

### 2.2 通用响应格式

成功响应：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

失败响应：
```json
{
  "code": 500,
  "msg": "错误信息",
  "data": null
}
```

---

### 2.3 错误码说明
| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录 / Token 失效 |
| 403 | 权限不足 |
| 500 | 服务器异常 |
| 600 | 业务异常 |

---

## 3. 删除策略说明
| 模块 | 删除方式 |
|------|-----------|
| 用户 | 软删除 |
| 文章 | 软删除 |
| 评论 | 软删除 |
| 点赞 | 硬删除 |
| 收藏 | 硬删除 |

说明：
- 软删除：数据库字段 `is_deleted = 1`，查询接口自动过滤  
- 硬删除：直接删除记录  

---

## 4. 用户模块

### 4.1 登录
POST /users/login

请求体：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 是 | 用户名 |
| password | String | 是 | 密码 |

响应字段：
```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "id": "18273645192837465",
    "name": "testUser",
    "avatarUrl": "https://xxx.com/avatar.png",
    "role": "USER",
    "token": "jwt-token"
  }
}
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 用户ID |
| name | String | 用户名 |
| avatarUrl | String | 用户头像 |
| role | String | 角色（USER / ADMIN） |
| token | String | JWT令牌 |

---

### 4.2 注册
POST /users

请求体：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 是 | 用户名 |
| password | String | 是 | 密码 |
| confirmPassword | String | 是 | 确认密码 |

响应字段：
```json
{
  "code": 200,
  "msg": "注册成功",
  "data": {
    "id": "18273645192837465"
  }
}
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 注册成功的用户ID |

---

### 4.3 获取用户信息
GET /users/{id}

路径参数：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 用户ID |

响应字段：
```json
{
  "code": 200,
  "msg": "查询成功",
  "data": {
    "id": "18273645192837465",
    "name": "testUser",
    "avatarUrl": "https://xxx.com/avatar.png",
    "createTime": "2024-05-20T10:30:00"
  }
}
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 用户ID |
| name | String | 用户名 |
| avatarUrl | String | 头像URL |
| createTime | String | 注册时间，LocalDateTime |

---

### 4.4 删除用户（软删除）
DELETE /users/{id}

权限：
- 仅 ADMIN 可操作  
- 非管理员返回 403

路径参数：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 用户ID |

---

## 5. 文章模块

### 5.1 创建文章
POST /articles

请求头：
- Authorization 必填

请求体：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | String | 是 | 文章标题 |
| description | String | 是 | 文章简介 |
| content | String | 是 | Markdown内容 |

说明：
- authorId 从 token 自动获取，无需前端传入

响应字段：
```json
{
  "code": 200,
  "msg": "创建成功",
  "data": {
    "id": "928374651234"
  }
}
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 新建文章ID |

---

### 5.2 获取文章详情
GET /articles/{id}

路径参数：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 文章ID |

响应字段：
```json
{
  "code": 200,
  "msg": "查询成功",
  "data": {
    "id": "928374651234",
    "authorId": "18273645192837465",
    "authorName": "张三",
    "title": "Java实战教程",
    "description": "这是一篇文章",
    "content": "# Markdown内容",
    "thumbCount": 10,
    "collectCount": 5,
    "viewCount": 100,
    "createTime": "2024-05-20T10:30:00",
    "updateTime": "2024-05-20T11:00:00"
  }
}
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 文章ID |
| authorId | String | 作者ID |
| authorName | String | 作者名称 |
| title | String | 标题 |
| description | String | 简介 |
| content | String | Markdown内容 |
| thumbCount | Integer | 点赞数 |
| collectCount | Integer | 收藏数 |
| viewCount | Integer | 浏览数 |
| createTime | String | 创建时间 |
| updateTime | String | 更新时间 |

---

### 5.3 分页获取文章列表
GET /articles?page=1&pageSize=10

查询参数：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | Integer | 否 | 当前页，默认1 |
| pageSize | Integer | 否 | 每页数量，默认10 |

响应字段：
```json
{
  "code": 200,
  "msg": "查询成功",
  "data": {
    "total": 100,
    "page": 1,
    "pageSize": 10,
    "records": [
      {
        "id": "928374651234",
        "title": "Java实战教程",
        "description": "文章简介",
        "authorId": "18273645192837465",
        "authorName": "张三",
        "thumbCount": 10,
        "collectCount": 5,
        "viewCount": 100,
        "createTime": "2024-05-20T10:30:00",
        "updateTime": "2024-05-20T11:00:00"
      }
    ]
  }
}
```

records 对象字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 文章ID |
| title | String | 标题 |
| description | String | 简介 |
| authorId | String | 作者ID |
| authorName | String | 作者名称 |
| thumbCount | Integer | 点赞数 |
| collectCount | Integer | 收藏数 |
| viewCount | Integer | 浏览数 |
| createTime | String | 创建时间 |
| updateTime | String | 更新时间 |

---

### 5.4 删除文章（软删除）
DELETE /articles/{id}

权限：
- 作者本人或 ADMIN

路径参数：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 文章ID |

---

| id | String | 是 | 文章ID |

### 5.6 收藏文章（硬删除结构）
- POST /articles/{id}/collect 收藏

请求头：
- Authorization 必填

路径参数：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 文章ID |

## 6. 评论模块

### 6.1 创建评论
POST /comments

请求头：
- Authorization 必填

请求体：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| articleId | String | 是 | 文章ID |
| content | String | 是 | 评论内容 |
| parentId | String | 否 | 父评论ID，顶级评论传 null |

响应字段：
```json
{
  "code": 200,
  "msg": "评论成功",
  "data": {
    "id": "5678912345"
  }
}
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 新建评论ID |

---

### 6.2 获取评论列表
GET /comments?articleId=928374651234&page=1&pageSize=10

查询参数：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| articleId | String | 是 | 文章ID |
| page | Integer | 否 | 当前页，默认1 |
| pageSize | Integer | 否 | 每页数量，默认10 |

响应字段：
```json
{
  "code": 200,
  "msg": "查询成功",
  "data": {
    "total": 20,
    "records": [
      {
        "id": "5678912345",
        "content": "这篇文章很有帮助！",
        "userId": "18273645192837465",
        "userName": "张三",
        "parentId": null,
        "createTime": "2024-05-20T12:00:00"
      }
    ]
  }
}
```

字段说明：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 评论ID |
| content | String | 评论内容 |
| userId | String | 评论者ID |
| userName | String | 评论者名称 |
| parentId | String | 父评论ID，顶级评论为 null |
| createTime | String | 评论时间 |

---

### 6.3 删除评论（软删除）
DELETE /comments/{id}

权限：
- 评论作者本人或 ADMIN

路径参数：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 评论ID |

---

## 7. 时间格式说明（重要）
- 后端使用 Java LocalDateTime  
- JSON 格式示例：
```
2024-05-20T10:30:00
```
- 格式：yyyy-MM-dd'T'HH:mm:ss  
- 默认使用服务器时区  
- 前端可根据需要转换为本地时间显示
